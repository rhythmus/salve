Before we move on to M7.2: Developer Tools — please explain what the " Visually interactive Developer Mode overlay" is, and which problem it should solve

*Viewed [development-roadmap.md](file:///Users/woutersoudan/Desktop/Salve/docs/development-roadmap.md) *

### Planner Response

The **Visually interactive Developer Mode overlay** is a "debug transparency layer" designed to solve the **"Black Box" Problem** inherent in culturally aware software.

Since Salve's greeting resolution is multi-dimensional (dependent on time, social context, anti-repetition memory, and linguistic transforms), it can be difficult for a developer to understand exactly *why* a specific greeting was chosen without digging through logs.

### The Problem It Solves
1. **Resolution Transparency**: When Salve says *"Moin, Herr Dr. Mustermann"* instead of *"Hallo Max"*, the developer needs to know if that happened because the "Academic Title" logic kicked in, because a specific "Temporal Event" (Morning) was active, or because the anti-repetition layer suppressed the standard fallback.
2. **Context Simulation (The "Time Travel" problem)**: Testing date-sensitive greetings (like Name-Days or Holiday-specific greetings) usually requires changing system clocks or hardcoding mock dates. The overlay provides a UI to "slide" through time or toggle social profiles (e.g., changing a user's gender or formality setting) to see how the greeting reacts in real-time.
3. **Linguistic Audit**: It allows developers to see the "Scoring" behind the scenes—showing which events were considered, which one "won," and which linguistic transforms (like the Greek Vocative) were applied to the names.

### Key Features of the Overlay:
* **The "Reasoning" Panel**: Displays the scores for all candidate greetings and highlights the winner.
* **Context Mocking**: Interactive inputs to override the `now` date, `formality`, or `AddressProfile` without touching the code.
* **Trace View**: A step-by-step breakdown of how the string was built:
    > `Seed Template` -> `Honorific Injection` -> `Name Transform` -> `Punctuation Cleanup`.

In M7.2, we will likely implement this as a lightweight UI component that can be toggled on/off in the browser, specifically for use in the `@salve/demo` and by developers integrating the library.
